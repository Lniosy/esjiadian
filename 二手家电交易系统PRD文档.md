## 一、产品概述

### 1.1 产品背景

随着绿色消费理念的深入人心和循环经济的蓬勃发展，二手家电交易市场呈现出巨大的潜力与活力。家电产品更新换代周期缩短，大量仍具使用价值的旧家电面临处置需求，同时，部分消费者出于经济实惠或临时性使用的考虑，对二手家电存在显著的购买需求。

然而，传统的线下二手市场、分类信息平台等交易模式存在诸多痛点：

1. **信息不透明**：商品描述真实性难以保障，实物与图片不符的情况时有发生
2. **交易信任度低**：买卖双方缺乏可靠的信用背书，资金安全与货物质量均存在风险
3. **流程不规范**：从沟通、议价到支付、物流，各个环节均依赖双方自行协商，效率低下且缺乏保障
4. **售后服务缺失**：交易完成后一旦出现问题，买家往往维权无门

### 1.2 产品定位

本产品定位为一个**专业化、垂直化的二手家电交易平台**，面向有二手家电买卖需求的个人用户，提供集信息发布、信用评估、在线沟通、安全支付、物流跟踪及售后评价于一体的全流程交易服务。平台聚焦于二手家电这一垂直领域，通过标准化的商品信息模板、AI智能推荐、强化型交易担保等特色功能，解决传统二手交易中的信任缺失和信息不对称问题。

### 1.2.1 目标用户群体

基于市场调研分析，本系统的主要目标用户群体包括：

1. **在校大学生**：经济预算有限，对二手家电有较高接受度，需求集中在小型家电（如电风扇、电饭煲、微波炉等）
2. **年轻租房群体**：临时性居住需求，追求性价比，对大家电（如冰箱、洗衣机、空调）有较强需求
3. **家电升级换代的卖家**：购置新家电后需要处置旧家电，希望盘活闲置资产
4. **追求实惠的家庭用户**：对品牌家电有需求但预算有限，愿意购买成色较好的二手商品

### 1.2.2 用户需求调研

通过对现有主流二手交易平台（闲鱼、转转）的深入分析和用户访谈，识别出以下核心需求：

| 需求类型 | 用户痛点 | 解决方案 |
|---------|---------|---------|
| 信息透明 | 商品描述不真实，图片与实物不符 | 标准化商品信息模板，强制填写关键参数 |
| 交易信任 | 缺乏信用背书，担心被骗 | 实名认证、信誉评价体系、担保交易 |
| 沟通效率 | 联系方式不统一，沟通不便 | 内置即时通讯，聊天记录可作凭证 |
| 支付安全 | 直接转账风险大 | 集成第三方支付，平台担保交易 |
| 物流跟踪 | 发货后无法追踪 | 物流接口集成，实时轨迹查询 |
| 售后保障 | 交易完成后维权困难 | 评价体系、纠纷处理机制 |

### 1.3 产品目标

#### 短期目标（课程设计阶段）

1. 完成系统核心功能模块的设计与开发
2. 实现用户注册登录、商品发布浏览、订单管理等基础交易流程
3. 构建基本的信用评价体系
4. 完成即时通讯功能的实现
5. 系统能够稳定运行，满足基本的二手家电交易需求

#### 长期目标（产品迭代方向）

1. 引入AI智能推荐算法，提升用户体验和交易转化率
2. 集成第三方支付和物流接口，实现完整的交易闭环
3. 建立完善的纠纷处理机制和售后服务体系
4. 扩展移动端应用，覆盖更多用户场景

### 1.4 项目意义

#### 社会意义

- 推动"物尽其用"的绿色消费理念
- 延长家电生命周期，减少电子垃圾
- 符合可持续发展战略，促进资源循环利用

#### 经济意义

- 激活二手市场存量资源
- 为卖家盘活闲置资产，创造额外收益
- 为买家降低购置成本，实现经济实惠

#### 实践意义

- 为二手商品交易的数字化转型提供可行技术方案
- 对探索其他品类二手物品的线上交易模式具有参考价值

## 二、用户角色与权限

### 2.1 用户角色定义

系统包含三类核心用户角色：买家用户、卖家用户、系统管理员。一个用户可以同时具备买家和卖家双重身份。

#### 2.1.1 买家用户

**角色描述**：有二手家电购买需求的注册用户，可浏览商品、与卖家沟通、下单购买、评价反馈。

**核心诉求**：
- 快速找到符合需求的二手家电
- 获取真实、全面的商品信息
- 与卖家便捷沟通，了解商品细节
- 安全、便捷地完成交易支付
- 实时了解订单和物流状态
- 交易完成后进行评价反馈

#### 2.1.2 卖家用户

**角色描述**：有二手家电出售需求的注册用户，可发布商品、管理店铺、处理订单、与买家沟通。

**核心诉求**：
- 便捷地发布和管理商品信息
- 展示个人信誉，获取买家信任
- 及时接收和处理买家订单
- 与潜在买家高效沟通
- 快速完成发货和物流更新

#### 2.1.3 系统管理员

**角色描述**：平台运营管理人员，负责系统管理、内容审核、纠纷处理、数据分析等工作。

**核心诉求**：
- 高效管理平台用户和内容
- 维护平台秩序，处理违规行为
- 及时处理交易纠纷
- 获取平台运营数据，支持决策

## 三、功能需求详细说明

### 3.1 用户管理模块

#### 3.1.1 用户注册

**功能描述**：新用户通过手机号或邮箱完成账号注册。

**业务规则**：
1. 支持手机号注册和邮箱注册两种方式
2. 手机号/邮箱需进行格式校验和唯一性校验
3. 密码需满足安全要求（长度≥8位，包含字母和数字）
4. 需完成验证码验证后方可注册成功
5. 注册成功后自动创建用户基础信息记录

**输入项**：
| 字段名 | 类型 | 必填 | 校验规则 |
|-------|------|-----|---------|
| 手机号/邮箱 | String | 是 | 格式校验、唯一性校验 |
| 密码 | String | 是 | 长度≥8位，含字母和数字 |
| 验证码 | String | 是 | 6位数字，5分钟内有效 |
| 用户协议 | Boolean | 是 | 必须勾选同意 |

**输出项**：注册成功提示，跳转至登录页面

#### 3.1.2 用户登录

**功能描述**：已注册用户通过账号密码或验证码登录系统。

**业务规则**：
1. 支持账号密码登录和验证码登录两种方式
2. 登录失败超过5次，账号锁定30分钟
3. 登录成功后生成Token，有效期7天
4. 支持记住登录状态功能

**输入项**：
| 字段名 | 类型 | 必填 | 校验规则 |
|-------|------|-----|---------|
| 手机号/邮箱 | String | 是 | 已注册账号 |
| 密码/验证码 | String | 是 | 正确的密码或验证码 |

**输出项**：登录成功，跳转至首页；Token信息存储

#### 3.1.3 实名认证

**功能描述**：用户提交实名认证信息，提升账号可信度。

**业务规则**：
1. 实名认证为可选功能，认证后可获得卖家权限
2. 需提交真实姓名、身份证号
3. 身份信息需与手机号实名信息一致
4. 认证通过后账号显示认证标识
5. 每个身份证只能认证一个账号

**输入项**：
| 字段名 | 类型 | 必填 | 校验规则 |
|-------|------|-----|---------|
| 真实姓名 | String | 是 | 2-20个汉字 |
| 身份证号 | String | 是 | 18位身份证格式校验 |
| 手机号 | String | 是 | 与注册手机号一致 |

**输出项**：认证结果（通过/审核中/拒绝）

#### 3.1.4 个人信息管理

**功能描述**：用户管理个人基本信息、头像、收货地址等。

**业务规则**：
1. 可修改昵称、头像、个人简介
2. 可添加、编辑、删除收货地址
3. 可设置默认收货地址
4. 手机号和邮箱修改需验证原绑定信息

### 3.2 商品管理模块

#### 3.2.1 商品发布

**功能描述**：卖家用户发布二手家电商品信息。

**业务规则**：
1. 发布商品前需完成实名认证
2. 商品信息需填写标准化模板
3. 商品发布后进入待审核状态
4. 审核通过后商品上架展示
5. 每个用户最多同时上架50件商品

**商品信息模板**：
| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| 商品标题 | String | 是 | 5-50字，描述商品核心信息 |
| 商品分类 | Enum | 是 | 大家电/小家电/厨电/数码等 |
| 品牌 | String | 是 | 家电品牌名称 |
| 型号 | String | 是 | 具体型号 |
| 购买日期 | Date | 是 | 原购买时间 |
| 新旧程度 | Enum | 是 | 全新/99新/95新/9成新/8成新/7成新及以下 |
| 功能状况 | Enum | 是 | 完好/轻微瑕疵/需维修 |
| 维修历史 | String | 否 | 是否维修过，维修情况说明 |
| 商品描述 | Text | 是 | 详细描述商品情况，100-2000字 |
| 售价 | Decimal | 是 | 定价，≥1元 |
| 原价 | Decimal | 否 | 原购买价格，供参考 |
| 商品图片 | Image[] | 是 | 1-9张，支持jpg/png格式 |
| 商品视频 | Video | 否 | 可选，展示商品实际运行状态 |
| 所在地区 | String | 是 | 省市区三级联动 |
| 交易方式 | Enum[] | 是 | 同城自提/快递邮寄 |

**输出项**：商品ID，发布成功提示

#### 3.2.2 商品编辑

**功能描述**：卖家编辑已发布的商品信息。

**业务规则**：
1. 仅可编辑自己发布的商品
2. 已售出商品不可编辑
3. 编辑后需重新审核
4. 审核期间商品下架

#### 3.2.3 商品上下架

**功能描述**：卖家管理商品的上架/下架状态。

**业务规则**：
1. 卖家可主动下架商品
2. 下架商品不在列表展示，但保留数据
3. 可重新上架已下架商品
4. 违规商品被管理员强制下架

#### 3.2.4 商品浏览与搜索

**功能描述**：买家浏览和搜索商品列表。

**搜索方式**：
1. **关键词搜索**：支持商品标题、品牌、型号模糊匹配
2. **分类导航**：按商品分类层级浏览
3. **条件筛选**：
   - 价格区间
   - 新旧程度
   - 功能状况
   - 所在地区
   - 交易方式
4. **排序方式**：
   - 综合排序（默认）
   - 价格升序/降序
   - 发布时间
   - 销量

**展示信息**：
- 商品首图
- 商品标题
- 价格
- 新旧程度
- 所在地区
- 卖家信誉等级

#### 3.2.5 商品详情

**功能描述**：展示商品完整信息。

**展示内容**：
1. 商品图片轮播/视频
2. 商品基本信息（标题、价格、品牌、型号等）
3. 商品详细描述
4. 卖家信息（昵称、信誉、认证状态）
5. 交易方式说明
6. 相似商品推荐
7. 立即购买/加入购物车/联系卖家按钮

### 3.3 智能推荐模块

#### 3.3.1 首页推荐

**功能描述**：首页展示个性化推荐商品。

**推荐策略**：
1. 基于用户浏览历史推荐相似商品
2. 基于用户搜索关键词推荐相关商品
3. 基于用户收藏/购买记录推荐
4. 热门商品推荐（新用户或数据不足时）
5. 同城好物推荐

**推荐位置**：
- 首页轮播位：精选推荐
- 猜你喜欢：个性化推荐
- 附近好物：同城推荐

#### 3.3.2 相关推荐

**功能描述**：商品详情页展示相关商品推荐。

**推荐策略**：
1. 同品牌其他商品
2. 同分类其他商品
3. 相似价位商品
4. 同卖家其他商品

### 3.4 即时通讯模块

#### 3.4.1 在线聊天

**功能描述**：买卖双方实时在线沟通。

**技术实现**：基于WebSocket实现实时通讯

**功能特性**：
1. 支持文字消息发送
2. 支持图片消息发送
3. 支持商品卡片分享
4. 消息已读/未读状态
5. 历史消息记录
6. 消息时间戳显示

**业务规则**：
1. 登录用户才能发起聊天
2. 聊天记录保存90天
3. 支持举报违规消息
4. 聊天记录可作为交易凭证

#### 3.4.2 消息通知

**功能描述**：系统消息和交易消息通知。

**通知类型**：
1. 系统通知：审核结果、违规提醒等
2. 交易通知：订单状态变更、支付结果等
3. 聊天消息：新消息提醒

**通知方式**：
1. 站内消息
2. 浏览器推送通知
3. 短信通知（重要消息）

### 3.5 购物车模块

#### 3.5.1 购物车管理

**功能描述**：买家管理待购买商品。

**功能特性**：
1. 添加商品到购物车
2. 修改商品数量（二手商品数量通常为1）
3. 删除购物车商品
4. 选择商品进行结算
5. 购物车商品状态监控（下架提醒）

**业务规则**：
1. 购物车最多添加20件商品
2. 商品下架自动标记失效
3. 商品售出自动移除

### 3.6 订单管理模块

#### 3.6.1 订单创建

**功能描述**：买家创建购买订单。

**订单信息**：
| 字段名 | 类型 | 说明 |
|-------|------|------|
| 订单编号 | String | 系统生成，唯一标识 |
| 商品信息 | Object | 商品快照信息 |
| 买家信息 | Object | 买家ID、联系方式 |
| 卖家信息 | Object | 卖家ID、联系方式 |
| 收货地址 | Object | 详细收货地址 |
| 订单金额 | Decimal | 商品金额 |
| 运费 | Decimal | 快递费用 |
| 实付金额 | Decimal | 订单金额+运费 |
| 交易方式 | Enum | 同城自提/快递邮寄 |
| 买家留言 | String | 备注信息 |
| 订单状态 | Enum | 订单当前状态 |

**订单状态流转**：
```
待付款 → 待发货 → 待收货 → 待评价 → 已完成
   ↓        ↓        ↓
 已取消   已退款   已退款
```

#### 3.6.2 订单支付

**功能描述**：买家完成订单支付。

**支付方式**：
1. 支付宝支付（集成支付宝API）
2. 微信支付（集成微信支付API）
3. 模拟支付（开发测试环境）

**业务规则**：
1. 订单创建后30分钟内未支付自动取消
2. 支付成功后订单状态变更为待发货
3. 支付失败可重新支付

#### 3.6.3 订单发货

**功能描述**：卖家填写发货信息。

**发货信息**：
| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| 物流公司 | Enum | 是 | 选择快递公司 |
| 物流单号 | String | 是 | 快递单号 |
| 发货备注 | String | 否 | 备注信息 |

**业务规则**：
1. 发货后订单状态变更为待收货
2. 系统自动推送发货通知给买家
3. 物流信息可实时查询

**物流接口集成**：
系统支持对接主流物流公司API，实现物流信息自动查询和推送：

1. **支持的物流公司**：
   - 顺丰速运
   - 中通快递
   - 圆通速递
   - 韵达快递
   - 申通快递
   - 邮政EMS
   - 京东物流

2. **物流API功能**：
   - 电子面单生成：自动生成快递面单
   - 运单轨迹查询：实时获取物流轨迹
   - 物流状态推送：物流状态变更时主动推送通知

3. **技术实现**：
   - 对接快递100/快递鸟等第三方物流API
   - 定时任务同步物流轨迹信息
   - 物流异常（如滞留、退回）自动告警

#### 3.6.4 订单收货

**功能描述**：买家确认收货。

**业务规则**：
1. 发货后7天自动确认收货
2. 买家可提前确认收货
3. 确认收货后订单状态变更为待评价
4. 支持申请退款/退货

#### 3.6.5 订单取消与退款

**功能描述**：取消订单或申请退款。

**业务规则**：
1. 待付款状态可直接取消
2. 待发货状态可申请退款，需卖家同意
3. 已发货状态可申请退货退款
4. 退款成功后恢复商品库存

### 3.7 评价管理模块

#### 3.7.1 评价发布

**功能描述**：买家对已完成订单进行评价。

**评价内容**：
| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| 评分 | Integer | 是 | 1-5星评分 |
| 评价内容 | String | 是 | 10-500字评价描述 |
| 评价图片 | Image[] | 否 | 0-6张图片 |
| 标签 | String[] | 否 | 预设标签选择 |

**业务规则**：
1. 订单完成后15天内可评价
2. 每个订单只能评价一次
3. 评价后不可修改删除
4. 评价发布后更新卖家信誉分

#### 3.7.2 评价展示

4. 评价标签云

## 五、技术架构

### 5.1 技术选型

#### 5.1.1 后端技术栈

| 技术 | 版本 | 用途 |
|-----|------|-----|
| SpringBoot | 2.7.x | 后端核心框架 |
| Spring Security | 2.7.x | 安全认证框架 |
| MyBatis-Plus | 3.5.x | ORM框架 |
| MySQL | 8.0 | 关系型数据库 |
| Redis | 7.0 | 缓存数据库 |
| WebSocket | - | 实时通讯 |

**各技术组件在本系统中的作用**：

1. **SpringBoot**：
   - 快速构建RESTful API：提供用户管理、商品CRUD、订单处理等功能的API接口，供前端Vue调用
   - 简化配置：自动配置机制减少繁琐的XML配置
   - 内置服务器：集成Tomcat，简化部署流程
   - 生态完善：丰富的Starter依赖，便于集成其他组件

2. **MySQL**：
   - 存储结构化数据：存储所有需要长期保存且关系明确的数据
   - 用户数据：用户账号、认证信息、收货地址等
   - 商品数据：商品信息、分类、图片等
   - 交易数据：订单、评价、聊天记录等

3. **Redis**：
   - 缓存热点数据：缓存频繁访问但不常变化的数据（如首页商品分类、热门商品列表），大幅减轻MySQL压力，提升系统响应速度
   - 存储验证码：存储短信/邮箱验证码，并设置短时自动过期
   - 分布式Session：支持分布式部署场景下的会话共享
   - 消息队列：简单的消息推送场景

4. **WebSocket**：
   - 实现即时通讯：让买家和卖家可以在商品详情页或订单页面进行实时文字聊天、发送图片，询价沟通
   - 消息实时推送：订单状态变更、系统通知等实时推送
   - 低延迟通信：全双工通信，避免轮询带来的资源浪费

#### 5.1.2 前端技术栈

| 技术 | 版本 | 用途 |
|-----|------|-----|
| Vue.js | 3.x | 前端框架 |
| Vue Router | 4.x | 路由管理 |
| Pinia | 2.x | 状态管理 |
| Element Plus | 2.x | UI组件库 |
| Axios | 1.x | HTTP请求库 |

**各技术组件在本系统中的作用**：

1. **Vue3**：
   - 构建单页面应用：创建流畅、无需重复加载页面的用户体验，适用于商品浏览、搜索、聊天等复杂交互场景
   - 组件化开发：将页面拆分为可复用的组件（如商品卡片、评价列表、聊天窗口），提高代码的可维护性和开发效率
   - 响应式数据绑定：数据与视图自动同步，简化商品列表更新、实时消息显示等功能的开发逻辑

2. **Element Plus**：
   - 提供丰富的UI组件：表单、表格、弹窗、导航等
   - 统一的设计风格：保证界面美观一致
   - 响应式布局：适配不同屏幕尺寸
