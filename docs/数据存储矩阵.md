# 数据存储矩阵（毕设答辩版）

## 说明
- 分类标准：`must`（必须落库）、`should`（建议落库）、`temp`（缓存/临时）、`forbidden`（禁止落库）
- 基准规则来源：`graduation-demo-system-rules`

## 矩阵
| 模块 | 关键数据 | 分类 | 存储位置 | 说明 |
|---|---|---|---|---|
| 认证与账号 | `sys_user`、角色、实名状态 | must | MySQL | 身份与权限真值 |
| 认证与账号 | JWT 黑名单/短期会话态 | temp | Redis | 可重建短期状态 |
| 认证与账号 | 验证码 | temp | Redis | TTL 过期数据 |
| 用户中心 | 地址簿 `user_address` | must | MySQL | 订单履约必要数据 |
| 商品 | `product`、`product_image`、审核状态 | must | MySQL | 核心交易对象 |
| 购物车 | `cart` | must | MySQL | 交易前状态需保留 |
| 订单交易 | `order_info`、`order_item`、状态机 | must | MySQL | 业务真值与追责依据 |
| 支付 | `payment_record` | must | MySQL | 支付状态审计依据 |
| 物流 | `logistics_record`、`logistics_track` | must | MySQL | 履约状态审计依据 |
| 退款售后 | `refund_request`、`dispute_case` | must | MySQL | 纠纷裁决依据 |
| 聊天通知 | `chat_message`、`notification`、`chat_message_report` | must | MySQL | 沟通凭证与举报凭证 |
| 评价信誉 | `evaluation`、卖家信誉分 | must | MySQL | 信誉体系核心数据 |
| 管理审计 | `operation_log`、`monitor_alert_event` | must | MySQL | 关键操作和告警处理可追溯 |
| 推荐 | `recommend_trace` | should | MySQL | 推荐策略持续优化数据 |
| 监控 | 实时指标快照 | temp | 内存 | 运行态监控值 |
| 文件上传 | 商品图/店铺图/聊天图/评价图/退款凭证图 | should | 本地文件目录 + MySQL URL | 统一走 `/api/files/upload/image`，文件本体落 `uploads`，业务表仅保存 URL 引用 |
| 安全敏感 | 密码明文、密钥明文、完整Token明文 | forbidden | - | 禁止持久化 |

## API-表-页面追踪示例
- 订单创建：`POST /api/orders` -> `order_info/order_item` -> `/orders`
- 商品审核：`POST /api/admin/products/reviews/{id}/approve|reject` -> `product` -> `/admin`
- 违规处理：`POST /api/admin/chat-reports/{id}/resolve` -> `chat_message_report/operation_log` -> `/admin`
